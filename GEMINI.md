## 1. 專案概覽
**ChatStar** 是一款 PC 端桌面應用程式，旨在透過 **Gemini Vision AI** 分析螢幕上的對話截圖，自動產生符合情境的繁體中文回覆建議。

- **核心目標**：自動化對話分析、連貫性脈絡理解、三風格回覆生成。
- **主要技術棧**：
  - **語言**：Python 3.12+
  - **UI 框架**：PyQt6 (用於桌面視窗與選取遮罩)
  - **AI SDK**：`google-genai` (Gemini-1.5-Flash)
  - **後端 API**：FastAPI (背景服務)
  - **資料庫**：PostgreSQL (透過 SQLAlchemy) + Pinecone (向量檢索)
  - **依賴管理**：`uv`

## 3. 開發規範與慣例

### 3.1 環境管理 (強制)
- **必須使用 `uv`** 進行套件安裝與環境管理。
- 禁止在全域環境執行 Python，應始終透過 `uv run` 或在 `.venv` 中執行。
- 安裝新套件請使用：`uv add <package>`。

### 3.2 程式碼風格
- **命名規範**：變數與函式使用 `snake_case`，類別使用 `PascalCase`。
- **UI 邏輯**：PyQt 視窗應繼承自對應的組件類別，並使用 `pyqtSignal` 進行跨執行緒溝通。
- **非同步處理**：長時間運行的任務（如截圖掃描、AI 呼叫）必須放在 `QThread` 中，避免阻塞 UI 主執行緒。

### 3.3 AI 提示詞 (Prompts)
- 所有的 Prompt 定義皆位於 `core/ai_provider.py`。
- 提取對話時採用「由下往上」策略，確保最新訊息的優先權。
- 回覆建議固定產生「正式」、「輕鬆」、「簡短」三種風格。

## 4. 建置與執行指令

### 啟動應用程式
```bash
uv run main.py
```

### 資料庫遷移與初始化
- 資料庫連線字串定義在 `.env` 的 `DATABASE_URL`。
- 首次運行會自動透過 `main.py` 中的 `SessionLocal` 與 `Base` 檢查。

### API Key 配置
- 編輯 `.env` 文件，加入 `GEMINI_API_KEY=your_key_here`。

## 5. 核心邏輯流程
1. **選取區域**：`SelectionWindow` 獲取座標。
2. **啟動掃描**：`ScreenScanner` 每 2 秒截圖一次。
3. **工作記憶**：`MemoryManager` 接收提取出的文字，維護 `max_window=8` 的對話視窗。
4. **AI 分析**：`GeminiProvider` 結合圖片與記憶 context，產出 JSON 格式建議。
5. **更新 UI**：透過信號將回覆內容傳遞至 `MainWindow` 顯示。
